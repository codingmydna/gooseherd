## Pick of the week

**Give `/orch` and `/arena` a shared, scriptable multi-verifier primitive instead of one fixed reviewer/judge.**

Right now both commands hardcode a single arbiter: `/orch`'s reviewer both judges completion *and* decides the exit condition, and `/arena`'s judge is a single model producing one ranking. Claude Code's Workflow tool demonstrates the alternative — an "adversarial verify" pattern where a claim survives only if a majority of N independently-prompted verifiers fail to refute it, cutting single-judge false-positive risk.

Ready-to-run `/orch` task:

> Add an optional `--verify-panel=N` flag to `/orch`. When set, the final reviewer VERDICT is replaced by N independent reviewer-role calls (same model/role, told to try to find a reason the implementation is incomplete or wrong), and the cycle only passes if a majority (`> N/2`) return APPROVE. Default N=1 (today's behavior, unchanged). Add an integration test that runs a cycle with `--verify-panel=3` against a seeded implementation with one deliberate bug and asserts the cycle does *not* pass (i.e. at least 2/3 reviewers catch it). Success criterion: `cargo test` passes, including the new test, and `--verify-panel=1` (default) produces byte-identical ledger output to the current unflagged run on the existing orchestrate test fixtures.
